Documentation

Everything you need to know about setting up BloxOLite products.

Introduction

MAES (Moderation and Enforcement System) is a admin framework built for Roblox developers who want full control over their moderation features. It’s fast, extensible, and supports full integration with your Discord ban logging.

Installation

Follow these steps to set up MAES in your game:

  1. Download MAES from the official Discord..
  2. Place the Server folder into ServerScriptService.
  3. Edit your BloxOLite module inside ServerScriptService to match your group roles and permissions.

General Configuration

return {
  ["General_Settings"] = {
    ["prefix"] = ";", -- Prefix used in game to execute commands
    ["Hotkey"] = "h", -- key that shows the UI
    ["BanData"] = data:GetDataStore("BanData123"), -- The ban data
    ["Modlogs"] = data:GetDataStore("ModerationLogs123"), -- The ModLogs Data
    ["WebHookUrl"] = "https://discord.com/api/webhooks/...", -- Discord Webhook for ban logging
		  
    ["BanPlaceEnabled"] = true, -- If the ban place is enabled or not
    ["BanPlaceID"] = "1234567890", -- Ban place ID
  },
}

Rank & Permission Settings

return {
  ["Guest"] = { -- Rank Name
    ["Rank_Needed"] = 0, -- Group rank needed
    ["Enabled"] = true, -- Rank Enabled
    ["Execute_Code"] = false, -- can it use 'runlua'
    ["Usable_Commands"] = { -- Command this rank+ can use
      "panel", "cmds", "clear", "help", "credits" -- command Names
    },
  },
}

Ranks stack — meaning higher roles automatically inherit lower-rank permissions.

Security & Roblox Settings

Ensure these features are enabled:

Allow HTTPS Requests = true (for Remote Events)
Enable API Services = true (for DataStores)

Without these, MAES will not be able to communicate with Discord.

Custom Commands

To add custom commands create a new module script in one of the command folders then add the code in.

local ReplicatedStorage = game:GetService("ReplicatedStorage")
local RepFolder = ReplicatedStorage:WaitForChild("NameHereFolder")
local messages = require(RepFolder:WaitForChild("ClientMessages"))
local CommandStatusEvent = RepFolder:WaitForChild("CommandStatusEvent")

return {
	Name = "CommandName", 
	Aliases = {},
	Description = "Description of what the command does.",
	Enabled = true,
	Function = function(player, args, Deps)
       -- command Function
		return true
	end
}

Then insert the Name, Description, Function also make sure to add the command name to a rank in BloxOLite_Config.

Anti-Cheat Docs

BloxOLite Anti-Cheat protects you game from exploits with regular updates from developers.

Installation

Follow these steps to set up Anti-Cheat in your game:

  1. Download/buy Anti-Cheat from the official Discord Server.
  2. Insert the file into your game.
  3. Drag AC_Manager into ServerScriptService and Anti-CheatClient into StarterPlayerScripts.
  4. Put the AntiCheat folder into ReplicatedStorage

General Configuration

  • To access the config open ReplicatedStorage -> AntiCheat -> Config and open the module to start configuring it to your liking.

    • ModCall Docs

    BloxOLite ModCall allows players in your game to call a staff memeber for help.

    Installation

    Follow these steps to set up ModCall in your game:

    1. Download/buy ModCall from the official Discord Server.
    2. Insert the file into your game.
    3. Drag ModCall into ServerScriptService and ModUI into StarterGui.
    4. Put the BloxOLiteRemotes folder into ReplicatedStorage
    5. On line 41 you will see Config.WebHookUrl = "https://discord.com/api/webhooks/..." replace it with you webhook url.

    General Configuration

  • To access the config open ServerScriptService -> ModCall -> settings and open the module to start configuring it to your liking.
