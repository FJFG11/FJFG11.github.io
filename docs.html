Getting Started

BloxOLite is a lightweight moderation toolkit for Roblox. This guide helps you add it to your game, configure permissions, and run commands.

Installation

  1. Download the package from GitHub Releases.
  2. Open Roblox Studio and insert the Module into ServerScriptService.
  3. Edit Settings.lua or the ModuleScript named Settings to set your group IDs and permissions.

Settings

The Settings module controls roles, DataStores, and command permissions. Example:

// Settings sample (simplified)
return {
  GroupId = 1234567,
  Roles = {
    Owner = { "Kick", "Ban", "ManageSettings" },
    Staff = { "Kick", "Warn" }
  },
  DataStores = {
    Punishes = "BloxOLite_PunishLog"
  }
}

Command Reference

Run moderation commands through the UI or chat console. A few common commands:

CommandDescription
!kick <player> <reason>Kicks a player immediately.
!ban <player> <duration> <reason>Bans a player for a duration (supports m/h/d).
!debughistory <player>Shows punishment history for a player.

API Integration

Emit logs to an external service or use the built-in JSON events for auditing.

{
  "event": "PlayerBanned",
  "player": "User123",
  "moderator": "Admin456",
  "reason": "Exploiting"
}

FAQ

Is BloxOLite safe for production?

Yes — it uses Roblox DataStores responsibly and provides configurable permissions for staff workflows.
